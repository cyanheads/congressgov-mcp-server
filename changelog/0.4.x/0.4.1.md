# 0.4.1 — 2026-06-30

Ships the GHCR container image for the 0.4.0 `congressgov_search_bills` mirror feature — 0.4.0 published to npm, the MCP Registry, and GitHub, but its multi-arch image failed to build. Packaging only; no source or runtime behavior change.

## Fixed

- **Container image build** — the 0.4.0 multi-arch GHCR build failed because `better-sqlite3`'s `node-gyp` postinstall ran during `bun install` in the toolchain-less `oven/bun` base image. The build-stage `bun install` now passes `--ignore-scripts`, matching the production stage; the image runs under Bun and uses `bun:sqlite`, so the native `better-sqlite3` binding is never loaded or needed in the container. ([#40](https://github.com/cyanheads/congressgov-mcp-server/issues/40))

## Changed

- **`better-sqlite3`** — promoted from an optional peer dependency to a regular `dependency` (`^12.11.1`); `trustedDependencies` removed. npm/Node consumers of the opt-in mirror now install it automatically; Bun continues to use its built-in `bun:sqlite` at runtime.
