# 0.4.0 — 2026-06-30

## Added

- **`congressgov_search_bills`** — keyword search over bill titles and CRS summary text, the discovery path the Congress.gov API structurally lacks (it exposes no keyword search). Backed by a local SQLite + FTS5 mirror; returns BM25-ranked matches, each with the derived bill id (`congress/billType/billNumber`) for a follow-up `congressgov_bill_lookup` call and a truncated summary preview, narrowable by optional `congress`, `billType`, and `originChamber` filters. Opt-in and off by default — registered via `disabledTool()` (visible but uncallable) until `CONGRESS_MIRROR_ENABLED=true`, so no existing tool changes. ([#40](https://github.com/cyanheads/congressgov-mcp-server/issues/40))
- **`CongressMirrorService`** (`src/services/congress-mirror/`) — a local FTS5 mirror of bill title + CRS summary built on the framework `MirrorService`, ingesting through the existing `CongressApiService` (bill list + `/summaries`). No live-API fallback: a mirror that has not finished its initial build returns an empty result with a notice, not an error.
- **Mirror lifecycle scripts** — `mirror:init` (full out-of-band build), `mirror:refresh` (incremental re-scan from the last `updateDate` checkpoint), and `mirror:verify` (readiness + SQLite integrity check), sharing a `scripts/_mirror-context.ts` bootstrap. Under HTTP transport, `CONGRESS_MIRROR_REFRESH_CRON` schedules an in-process incremental refresh; the mirror is never built or refreshed on server startup.
- **Four optional mirror env vars** — `CONGRESS_MIRROR_ENABLED` (default `false`), `CONGRESS_MIRROR_PATH` (default `.mirror/bills.sqlite3`), `CONGRESS_MIRROR_REFRESH_CRON` (unset), and `CONGRESS_MIRROR_CONGRESSES` (default: current congress + the one prior). None required — the standard deploy stays config-free.
- **`Dockerfile`** — mirror-CLI copy stanza so `docker exec <container> bun run mirror:init` / `mirror:refresh` / `mirror:verify` work against a deployed image; copies the lifecycle scripts and emits a runtime `tsconfig.json` resolving the `@/` alias against `./dist`.

## Dependencies

- **`node-cron`** `^4.4.1` added — in-process incremental-refresh scheduler.
- **`better-sqlite3`** `^12.0.0` added as an optional peer dependency (`^12.11.1` devDependency) — Bun uses its built-in `bun:sqlite` at runtime; `better-sqlite3` covers the Node install path.
