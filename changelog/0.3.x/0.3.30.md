# 0.3.30 — 2026-06-11

Framework adoption release — picks up the `@cyanheads/mcp-ts-core` 0.10.x line. No tool, resource, or prompt surface changes; behavior is unchanged.

## Added

- **Server identity pair** — `createApp()` now sets `name` and `title` to `congressgov-mcp-server`, so the unscoped machine name shows on every client surface instead of the npm-scoped package name.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner wired into the `bundle` script: runs `mcpb clean` then strips `node_modules/**` agent-doc entries (`skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach.
- **Dockerfile `HEALTHCHECK`** — bun-native `fetch` against `/healthz` every 30s (no `curl`/`wget` dependency on the slim image).
- **Dockerfile writable data dirs** — pre-creates `/usr/src/app/.cache` and `/usr/src/app/.mirror` owned by the `bun` runtime user, so on-disk SQLite stores open cleanly for the non-root user.
- **`techniques` skill** — catalog of reusable response/data-shaping patterns (first reference: outline-on-overflow).

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.21` → `^0.10.6`. Notable upstream changes consumed on upgrade: env-boolean parsing tightened to Zod `stringbool` (rejects unrecognized values at startup); `ctx.sample`, sampling types, and `RootsRegistry` removed (SEP-2577 — this server used none); `ctx.elicit` now wired on the wire (this server presence-gates none); DataCanvas SQL gate fails closed on non-SELECT and denies `pragma_*`.
- **`.codex-plugin/plugin.json`** — `interface.displayName` de-scoped from `@cyanheads/congressgov-mcp-server` to `congressgov-mcp-server`.
- **Dockerfile** — `ARG APP_VERSION` stamps `org.opencontainers.image.version`; the release `docker buildx` passes `--build-arg APP_VERSION`.
- **`.mcpbignore`** — root-anchored every dev-dir pattern (`/skills/`, `/.claude/`, `/.agents/`, `/Dockerfile`, …) so they can't strip nested `node_modules` runtime paths.
- **`scripts/lint-packaging.ts`** — adds bundle-content guards (anchoring, critical-path, post-bundle agent-doc) and `name`/`title`/`display_name` identity checks against the unscoped package name.
- **`scripts/check-framework-antipatterns.ts`** — adds the `coerce-boolean-env-flag` rule (flags `z.coerce.boolean()` in `src/`, recommends `z.stringbool()`); the scanner now skips comment lines.
- **Framework skills synced** to the 0.10.x bodies (`api-canvas`, `api-context`, `api-linter`, `api-mirror`, `api-testing`, `api-workers`, `polish-docs-meta`, `orchestrations`, and the tool/resource/prompt/service scaffolds).
- Dependencies: `@types/node` `^25.9.1` → `^25.9.3`.
