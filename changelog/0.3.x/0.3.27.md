# 0.3.27 ‚Äî 2026-05-31

## Added

- **`filter` param on `committee_lookup` `operation: 'list'`** ‚Äî resolves a committee name to `systemCode` client-side. Fetches the full chamber set, applies all-token primary match (order-independent, handles partials), falls back to best-token bigram-Dice similarity when the primary match is empty; fuzzy rows are labeled `approximate`. Never throws on no-match ‚Äî returns empty data with a notice to call `list` unfiltered. ([#38](https://github.com/cyanheads/congressgov-mcp-server/issues/38))

## Fixed

- **`getCommittee` / `getCommitteeSubResource` not-found errors** now carry the code shape (`hspw00` = chamber prefix + abbreviation + digits) and redirect to `operation: 'list'` with the optional `filter`, matching the pattern of sibling tools (`getMember`, CRS report, nominee batch). ([#37](https://github.com/cyanheads/congressgov-mcp-server/issues/37))
- **Whitespace guard in `committee_lookup` handler** ‚Äî a `committeeCode` containing whitespace (a name, not a code) is rejected before the API round-trip with a `validationError` pointing to `operation: 'list'`. ([#37](https://github.com/cyanheads/congressgov-mcp-server/issues/37))
