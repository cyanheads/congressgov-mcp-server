# 0.3.26 — 2026-05-31

## Added

- **`congressgov_roll_votes` `chamber` parameter** — accepts `'house'` (default, backward-compatible) or `'senate'`. Existing calls that omit `chamber` continue to hit the Congress.gov API unchanged. Resolves [#30](https://github.com/cyanheads/congressgov-mcp-server/issues/30).
- **`SenateVoteService`** — dedicated client for the Senate's official LIS roll-call XML feed (`senate.gov/legislative/LIS/roll_call_votes/`); used for all Senate operations (the Congress.gov API v3 exposes no Senate vote namespace).
- **Senate `list` operation** — browses roll call votes for a congress/session by fetching and client-paging the LIS session menu XML.
- **Senate `get` operation** — returns vote question, date, result, yea/nay/present/not-voting tallies, party breakdown (derived from the member roster — the feed publishes none), and associated bill/nomination/amendment metadata.
- **Senate `members` operation** — returns each senator's recorded position for a specific vote.
- **`src/services/senate-lis/parse.ts`** — pure XML-to-domain parsers backed by `fast-xml-parser`; fixture-tested against real LIS feed data.
- **`src/services/senate-lis/types.ts`** — Senate vote domain types.

## Changed

- **`format-helpers.ts`** — added Senate list, detail, and members renderers alongside existing House renderers.

## Dependencies

- `fast-xml-parser` added at `^5.8.0` (runtime dep; no new optional configuration required).
