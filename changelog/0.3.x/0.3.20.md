# 0.3.20 — 2026-05-20

## Added

- **`Search:` echo line on every list response** — list output now leads with `_Search: <scope> (filter=value, …)_` so the caller can confirm what was searched, even when results are empty. Renders the filters actually applied (congress, billType, date range, state, etc.).
- **ISO 8601 validation on `fromDateTime` / `toDateTime`** — `congressgov_bill_lookup` and `congressgov_bill_summaries` now reject malformed date-times up front with an actionable message, instead of forwarding them and surfacing the opaque upstream 400.
- **Tighter output schemas across all ten tools** — list operations declare `data` + `pagination` (`count`, `nextOffset`); detail operations declare the entity key (`bill`, `member`, `committee`, `vote`, `nomination`, `report`, `law`). Replaces the prior catch-all `passthrough` declarations.

## Changed

- **`congressgov_roll_votes` `order='recent'` is strictly newest-first** ([#27](https://github.com/cyanheads/congressgov-mcp-server/issues/27)) — fetches the full session, sorts by `updateDate` descending, then pages. Late-edited votes now land at the top instead of getting buried by the upstream's opaque insertion order. Typical sessions cost 1–3 upstream requests; subsequent pages are dispatched in parallel.
- **`congressgov_senate_nominations` sub-resources render properly again** ([#26](https://github.com/cyanheads/congressgov-mcp-server/issues/26)) — `actions` rows show date + text + type, `committees` rows show committee name + activities, `nominees` rows show full name + ordinal + state, `hearings` rows show citation + chamber + date. Fixes the v0.3.19 regression where every sub-resource row rendered as bare `### N. Nomination` with all useful fields dropped.
- **Empty paginated pages are now distinguishable from "0 total"** ([#28](https://github.com/cyanheads/congressgov-mcp-server/issues/28)) — when the page is past the end of a non-empty result set, output now reads `_Page is empty — offset is past the end of N total items._` instead of just `**N results**` with no items.
- **`congressgov_committee_lookup` description documents `bills` sub-resource sparsity** ([#29](https://github.com/cyanheads/congressgov-mcp-server/issues/29)) — calls out that upstream omits titles on this endpoint and points callers at `congressgov_bill_lookup get` per row for titles and policy area.
- **`congressgov_enacted_laws` description rewritten** — drops the prior "for symmetry" framing of `get` and describes what the payload actually is: the origin bill record with the public/private law citation on its `laws` array.
- **Bill and CRS report detail views lead with a title-first header** — bill detail shows congress, chamber, policy area, dates, and latest action up top; CRS report detail shows type, status, version, dates, and authors up top. Replaces the prior raw field dump.
