# 0.3.14 — 2026-04-24

Maintenance release. Pulls in the full 0.6.x + 0.7.0 arc of `@cyanheads/mcp-ts-core` — landing page, SEP-1649 Server Card, stricter linter, new error helpers, skill/script sync infrastructure — and migrates this repo to the directory-based changelog convention the framework now recommends. No tool or resource behavior changes; every definition exercised clean via field-test.

## Added

- **Directory-based changelog** — migrated every historical release from the monolithic `CHANGELOG.md` into per-version files under `changelog/<major.minor>.x/<version>.md` (17 files across `0.1.x`, `0.2.x`, `0.3.x`). `CHANGELOG.md` is now an auto-generated navigation index. `changelog/template.md` is the pristine format reference. New `bun run changelog:build` / `bun run changelog:check` scripts wired in; `changelog:check` runs as a devcheck step that hard-fails on drift.
- **Landing page view-source integration** (`src/index.ts`) — HTTP-mode landing page now advertises this repo as `landing.repoRoot` and each definition carries a `sourceUrl` override pointing at its real file. The framework's default auto-derivation (`<repoRoot>/blob/main/src/mcp-server/<kind>/definitions/<kebab-of-name>.ts`) doesn't match this server's file layout because filenames strip the `congressgov_` name prefix — the inline `withSource()` helper supplies the actual path so view-source links resolve on GitHub.
- **Three new skills synced from the framework**: `api-linter` (lint-rule reference for `format-parity` / `schema-*` / `name-*` / `server-json-*` diagnostics), `security-pass` (8-axis MCP security audit), `release-and-publish` (post-wrapup ship workflow for npm / MCP Registry / GHCR).
- **`scripts/build-changelog.ts`** + **`scripts/check-skills-sync.ts`** synced from the framework (Phase C of the `maintenance` skill). Devcheck picks up `Skills Sync` and `Changelog Sync` steps as warnings/errors when mirrors drift.
- **GitHub issue templates** at `.github/ISSUE_TEMPLATE/` for bug reports, feature requests, and template config (labels auto-apply, PAT placeholder redaction guidance in the body).

## Changed

- **`@cyanheads/mcp-ts-core` `^0.5.3` → `^0.7.0`** — single jump across 18 releases. Framework changes adopted automatically (logic throws / framework catches, auto-instrumentation, startup definition lint, the recursive `describe-on-fields` rule, flattened ZodError messages in `McpError.message`, structured `data.issues`). No breaking changes. No handler or tool-schema edits required.
- **`@biomejs/biome` `^2.4.12` → `^2.4.13`**, **`vitest` `^4.1.4` → `^4.1.5`** — patch bumps via `bun update --latest`.
- **`scripts/devcheck.ts`**, **`scripts/lint-mcp.ts`**, **`scripts/tree.ts`** — resynced from the framework package. `devcheck` picks up new `Skills Sync` + `Changelog Sync` steps, a regex-escape security fix in the color helper, and an audit-connection-failure detection branch. `lint-mcp` now validates `server.json` + `package.json` alongside tool/resource/prompt definitions. `tree` collapses the new `changelog/<series>/` per-version files for readability.
- **Skills refreshed from the framework package** (Phase A of `maintenance`): `add-app-tool`, `add-prompt`, `add-resource`, `add-service`, `add-tool`, `api-context`, `api-utils`, `design-mcp-server`, `field-test`, `maintenance`, `polish-docs-meta`, `report-issue-framework`, `report-issue-local`, `setup`. Mirrors at `.claude/skills/` and `.agents/skills/` re-propagated (Phase B). Local `devcheck` + `migrate-mcp-ts-template` skills preserved.
- **`CLAUDE.md` + `AGENTS.md`** — adopted framework-authored template updates: `security-pass` slotted into the "What's Next?" sequence, skill table gained `api-linter` / `security-pass` / `release-and-publish` / `add-app-tool`, agent-directory note now points at the `maintenance` skill for auto-sync, Changelog section added describing the new directory-based format, checklist expanded with form-client optional-object guard and external-API-wrapping bullets (the latter particularly relevant for this Congress.gov wrapper — upstream sparsity, nullability, sparse-payload test coverage).
- **`package.json` scripts** — added `changelog:build` + `changelog:check`.
- **`docs/tree.md`** — regenerated to reflect the new `changelog/` layout and synced skill set.

## Notes

- Filed [cyanheads/mcp-ts-core#64](https://github.com/cyanheads/mcp-ts-core/issues/64) on a transient `ERR_UNSUPPORTED_NODE_MODULES_TYPE_STRIPPING` failure seen running `bun run test` on Node 25 — the framework's `./vitest.config` export resolves to a raw `.ts` file inside `node_modules`, which Node's type-stripper refuses to process. Workaround for this server today: `NODE_OPTIONS=--no-experimental-strip-types bun run test` if the error recurs. Tests pass cleanly as-is in our CI.
