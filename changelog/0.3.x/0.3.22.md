# 0.3.22 — 2026-05-28

## Changed

- **`@cyanheads/mcp-ts-core` `^0.9.6` → `^0.9.13`** — framework adoption across the 0.9.7–0.9.13 series.
- **`landing.requireAuth: false`** — explicit opt-in added to `src/index.ts` after 0.9.13 changed the default to auth-gated when `MCP_AUTH_MODE` is `jwt` or `oauth`; preserves public landing inventory for `none`-auth deployments.
- **`manifest.json`** — description updated to match `package.json`; `repository`, `homepage`, and `license` fields added; `CONGRESS_API_KEY` user_config entry gains `"default": ""`.
- **`package.json` keywords** — `bun`, `stdio`, `streamable-http` added; now surfaced via `GET /mcp` (0.9.12 framework feature).
- **`.env.example`** — `MCP_HTTP_MAX_BODY_BYTES` commented entry added (configurable 413 body cap, default 1 MiB).

## Security

- **`MCP_HTTP_MAX_BODY_BYTES` inbound body cap** (via framework 0.9.13) — oversized requests rejected with HTTP 413 before the MCP transport allocates. Default 1 MiB; set to `0` to disable.
- **HTTP session-init gate** (via framework 0.9.10) — stateful HTTP mode rejects non-`initialize` requests missing `Mcp-Session-Id` with HTTP 400.

## Fixed

- **Quieter expected-error logging** (via framework 0.9.10) — 401, 403, 400, and 404 responses now use `warning`-level logging instead of full stack traces.

## Dependency bumps

- `@cyanheads/mcp-ts-core` `^0.9.6` → `^0.9.13`
- `@biomejs/biome` `^2.4.15` → `^2.4.16`
