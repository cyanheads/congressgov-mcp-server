# 0.3.16 — 2026-05-08

Quality-of-life pass on tool output and error messages. No new tools, no API changes.

## Added

- **`tryNotFound` service helper** — wraps generic upstream 404s into descriptive `notFound` errors carrying every identifier the caller supplied. Bill / law / member / committee / vote / nomination / nominee batch / daily-record / CRS report / committee report endpoints now surface messages like `Bill HR 3076 not found in the 119th Congress.` instead of `Not found.`. Includes an English `ordinal()` helper (`119` → `119th`).
- **HTML → Markdown rendering for bill summaries** — `renderSummaryItem` in `format-helpers.ts` now converts upstream `<p>` / `<strong>` / `<em>` / anchor tags + named entities to structured Markdown. Previously stripped to plain text, losing paragraph breaks, emphasis, and links.
- **Empty-result recovery hint** — `renderList` appends `Try broadening the filters — widen the date range, drop the type/chamber/state constraint, or remove the date window entirely.` when no items match. Improves agent recovery on overly-narrow queries.
- **Shared `BILL_TYPE_CODES` constant** in `services/congress-api/types.ts` — single source for the `hr | s | hjres | sjres | hconres | sconres | hres | sres` enum, consumed by `bill-lookup.tool.ts`, `bill-summaries.tool.ts`, and `bill.resource.ts`. Removes three copies of the literal array.

## Changed

- **Field casing normalized to match the rest of the API surface:**
  - `congressgov_roll_votes` operation `members` — `bioguideID` → `bioguideId`.
  - `congressgov_crs_reports` operations `list` / `get` — `relatedMaterials[].URL` → `url`.
  - `congressgov_committee_reports` operation `list` — `cmte_rpt_id` → `cmteRptId`.
- **Resource params now strictly validated:**
  - `congress://bill/{congress}/{billType}/{billNumber}` — `congress` and `billNumber` regex `^\d+$`; `billType` constrained to `BILL_TYPE_CODES`.
  - `congress://committee/{committeeCode}` — regex `^[hsj][a-z0-9]{3,8}$` with `'h' (House), 's' (Senate), or 'j' (Joint)` hint.
  - `congress://member/{bioguideId}` — regex `^[A-Z]\d{6}$`.
- **`congress://committee/{committeeCode}` resource** now exposes top-level `name` (extracted from `history[0].officialName` / `libraryOfCongressName`) and `chamber` (inferred from the code's first letter) alongside the upstream payload.
- **`CongressDetail` type** corrected to upstream shape: `congress: number` → `number: number`; `endYear` / `startYear` are `string`; new optional `updateDate` and `url`. `current-congress.resource.ts` log statement now reads `congress.number`.
- **Tool / prompt descriptions tightened** — `congressgov_bill_summaries` now points one-bill queries at `congressgov_bill_lookup`; `congressgov_daily_record` lists the operation hierarchy explicitly; `congressgov_enacted_laws` moves `pub` / `priv` glossary into the `lawType` field describe; `congressgov_senate_nominations` lists operations upfront. `congressgov_bill_analysis` prompt adds `congressgov_committee_reports`; `congressgov_legislative_research` prompt adds an explicit bill-detail step.

## Fixed

- `getNominee` no longer shadows the new `ordinal()` helper — its `ordinal: number` parameter is now `ordinalNum`.
