# 0.3.25 — 2026-05-31

## Changed

- **`congressgov_roll_votes` `operation=members`** — member roster moved from `vote.results` to top-level `data[]`, matching the fleet-wide `{ data, pagination, totalCount, effectiveQuery }` envelope ([#36](https://github.com/cyanheads/congressgov-mcp-server/issues/36)). `vote` is now a sibling object (same record as `get` returns, minus `results`); `pagination`/`totalCount` count `data`. `operation=get` and `operation=list` are unchanged.
- **`formatVotes` members branch** — markdown output leads with a vote-context header (roll, congress/session, question, result, legislation), then `Members X–Y of N`, then the roster list. Vote context is still rendered inline — no extra round-trip needed.
