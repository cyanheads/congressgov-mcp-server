# 0.3.17 — 2026-05-16

Framework uptake release: `@cyanheads/mcp-ts-core ^0.8.19 → ^0.9.1` (15 intermediate releases, including the breaking 0.9.0 cut). No tool/resource surface changes; behavioral changes are limited to error-code mapping and the new `initialize` instructions payload.

## Added

- **Server-level `instructions`** ([mcp-ts-core#91](https://github.com/cyanheads/mcp-ts-core/issues/91)) — `createApp({ instructions })` in `src/index.ts` returns a one-paragraph orientation on every MCP `initialize` response. Spec-compliant clients forward it to the model as session-level context, replacing what previously would've leaked into every tool description. Covers domain (Congress.gov API v3), no-keyword-search semantics, and identifier conventions (`congress/billType/billNumber`, `bioguideId`, committee chamber-prefix codes).

## Changed

- **Tool input-validation errors use the `validationError` factory** across all 10 tool handlers (`bill-lookup`, `bill-summaries`, `committee-lookup`, `committee-reports`, `crs-reports`, `daily-record`, `enacted-laws`, `member-lookup`, `roll-votes`, `senate-nominations`). Previously `throw new Error(...)`, which the framework classified as `InternalError` (`-32603`). Now surfaces as JSON-RPC `ValidationError` (`-32602`) with structured `error.data` carrying the offending field names — proper error code for the failure mode and machine-actionable context for callers.
- **`CongressApiService.fetchResponse` drops stale status-code probing.** Was unwrapping `error.data.statusCode === 404 / 429` from a generic `ServiceUnavailable` envelope; framework now maps HTTP status → typed `McpError` directly (`404 → NotFound`, `429 → RateLimited`, `5xx → ServiceUnavailable/InternalError/Timeout`). Local catch keeps only the `RateLimited` rewrap — preserves the domain-specific quota message (`5,000 requests/hour`); 404 rewrapping continues at call sites via `tryNotFound()` with identifier-rich data.
- **`scripts/devcheck.ts` `bun outdated` parser fixed** (mirrors [mcp-ts-core#0.9.1](https://github.com/cyanheads/mcp-ts-core/blob/main/changelog/0.9.x/0.9.1.md)). `bun outdated` emits markdown rows (`| pkg | ... |`) with an empty leading cell, so `split('|')[0]` was always empty and the allowlist never matched. Now reads index `[1]` and strips the trailing `(dev|peer|prod|optional)` workspace marker.
- **`scripts/build-changelog.ts` summary cap `250 → 350` chars** (mirrors [mcp-ts-core#129](https://github.com/cyanheads/mcp-ts-core/issues/129)). 250 was clipping operator-relevant context from rollup summaries. `changelog/template.md` updated in lockstep.
- **Test mocks include `headers: new Headers()`** in `okJson` / `errorResponse` helpers — required because `fetchWithTimeout` reads response headers (status mapping, `Retry-After`). The CRS-outage 500 test now asserts `InternalError` instead of `ServiceUnavailable` to match the framework's tightened classification of HTML-bodied 500s.
- **Skills re-synced from framework** — refreshed `add-tool`, `api-auth`, `api-config`, `api-errors`, `api-linter`, `api-workers`, `design-mcp-server`, `field-test`, `polish-docs-meta`, `security-pass`, `tool-defs-analysis`, plus the bundled `polish-docs-meta/references/` README/package-meta templates. Mirrored to `.claude/skills` and `.agents/skills`.
- **README** — `OTEL_ENABLED` row now links to framework telemetry docs.
