# 0.3.19 — 2026-05-20

No surface removals; additive params default to existing behavior and format upgrades preserve previously-rendered fields.

## Added

- **`order: 'recent' | 'oldest'` on `congressgov_bill_lookup` list** ([#23](https://github.com/cyanheads/congressgov-mcp-server/issues/23)) — sorts by `updateDate`; defaults to `recent`. Implemented server-side via `?sort=updateDate desc|asc` on `/bill/{c}[/{t}]`. `ListBillsParams` gains optional `sort`.
- **`order: 'recent' | 'oldest'` on `congressgov_roll_votes` list** ([#23](https://github.com/cyanheads/congressgov-mcp-server/issues/23)) — Congress.gov ignores sort params on `/house-vote/{c}/{s}`, so `recent` (default) is implemented client-side via probe-count → fetch-tail → reverse. `offset` is interpreted in the reversed view; `offset=0` always returns the most recent page.
- **Chamber inference on `congressgov_committee_lookup`** ([#9](https://github.com/cyanheads/congressgov-mcp-server/issues/9)) — `get`, `bills`, `reports`, and `nominations` no longer require `chamber`. `inferChamberFromCode` derives it from the first letter of `committeeCode` (`h` → house, `s` → senate, `j` → joint); pass `chamber` explicitly to override. `list` still uses chamber as a filter.
- **Domain renderers replacing raw camelCase dumps** ([#18](https://github.com/cyanheads/congressgov-mcp-server/issues/18)) — `renderMemberDetail`, `renderRollVoteDetail`, `renderNominationListItem` / `renderNominationDetail`, `renderCommitteeListItem`, `renderCommitteeReportListItem`, `renderAmendmentItem`, `renderBillTextItem`, `renderBillSubresourceSummaryItem`. Each `formatXxx` exports route detail and list paths through the new renderers.
- **Parent-nomination partitioned-form hint** ([#13](https://github.com/cyanheads/congressgov-mcp-server/issues/13), [#14](https://github.com/cyanheads/congressgov-mcp-server/issues/14)) — `senate_nominations get` on a multi-part parent (e.g. `PN851`, `partNumber=00` with no `nominees` array) appends a callout in detail rendering. Sub-resource calls (`actions`/`committees`/`hearings`/`nominees`) against a bare parent attach an `emptyHint` that surfaces in the empty-result formatter, pointing callers at the partitioned form (`851-1`, `851-2`, …).
- **Opt-in `emptyHint` on `renderList`** ([#22](https://github.com/cyanheads/congressgov-mcp-server/issues/22)) — formatters now pass context-specific recovery text per call. Replaces the hardcoded "broaden the filters" advice that was irrelevant on most sub-resource operations (`cosponsors`, `nominees`, `actions`, `text`, …).

## Changed

- **`stripHtml` preserves paragraph and line breaks by default** ([#17](https://github.com/cyanheads/congressgov-mcp-server/issues/17)) — `<br>` → `\n`, `</p>` → `\n\n`; previously collapsed everything to one line. Compact-field-display callers pass `{ inline: true }` to keep the one-line behavior. Bill summaries now render with paragraph structure intact.
- **`formatMembers` dispatches per-row across mixed bill+amendment lists** ([#6](https://github.com/cyanheads/congressgov-mcp-server/issues/6)) — amendments (identified by `amendmentNumber`) render with chamber-aware heading (e.g. "Senate Amendment 1234") inferred from the URL path (`samdt`/`hamdt`) and latest-action metadata, instead of `### N. Untitled`.
- **`formatNominations` renders `nominationType` as a readable label** ([#16](https://github.com/cyanheads/congressgov-mcp-server/issues/16)) — `{isCivilian: true}` → "Civilian", `{isMilitary: true}` → "Military". Replaces the stringified inline-object dump on both list and detail paths.
- **`formatCommitteeReports` list uses citation-first heading** ([#19](https://github.com/cyanheads/congressgov-mcp-server/issues/19)) — surfaces `citation`, `congress`, `chamber`, `type`, `number`, `updateDate` (and `part` when ≠ `"1"`). Upstream omits `title` on list rows; this gives callers identifying metadata to drill in.
- **`renderCrsReportItem` omits the summary line when absent** ([#25](https://github.com/cyanheads/congressgov-mcp-server/issues/25)) — drops the `_Summary not available._` placeholder that previously appeared on every sparse row.
- **`pickBillListRenderer` detects bill-text and bill-subresource-summary shapes** ([#17](https://github.com/cyanheads/congressgov-mcp-server/issues/17), [#20](https://github.com/cyanheads/congressgov-mcp-server/issues/20)) — text rows render as `### N. <type>` (e.g. "Enrolled Bill") with each format URL as a labeled link; summaries render via `htmlToMarkdown` so `<p>` / `<strong>` survive.
- **`congressgov_bill_lookup` date-filter description corrected** ([#15](https://github.com/cyanheads/congressgov-mcp-server/issues/15)) — `fromDateTime` / `toDateTime` filter by the bill's `updateDate` (when Congress.gov last touched the record), not `latestAction.actionDate`. README updated in lockstep.
- **`congressgov_senate_nominations` description documents partitioned-form semantics** ([#13](https://github.com/cyanheads/congressgov-mcp-server/issues/13), [#14](https://github.com/cyanheads/congressgov-mcp-server/issues/14)) — multi-part parents (e.g. `PN851`) carry no own activity; actions/committees/hearings/nominees live on partitioned children. `get` on a parent that lacks a `nominees` array signals the partitioned form is needed.
- **`congressgov_enacted_laws` description clarifies value relative to `bill_lookup`** ([#24](https://github.com/cyanheads/congressgov-mcp-server/issues/24)) — `list` is the unique surface (filters bills by enactment status + law type, which `bill_lookup` can't); `get` mirrors `bill_lookup`'s bill detail. Points callers at `bill_lookup` as canonical for detail.

## Fixed

- **`congress-api-service.normalizeCrsReport`** ([#5](https://github.com/cyanheads/congressgov-mcp-server/issues/5), [#21](https://github.com/cyanheads/congressgov-mcp-server/issues/21)) — `relatedMaterials[]` dedupes by URL (upstream returns duplicates with mixed casing); the report's own `url` field gets `https://` prepended when it ships schemeless (`www.congress.gov/...`).
- **`congressgov_bill_lookup text` returns exactly `limit` items** ([#4](https://github.com/cyanheads/congressgov-mcp-server/issues/4)) — `/bill/{c}/{t}/{n}/text` consistently returns `limit+1`; `fetchList` truncates to the requested page size so pagination is consistent across the surface.
- **`congressgov_committee_lookup reports` returns ISO-8601 `updateDate`** ([#7](https://github.com/cyanheads/congressgov-mcp-server/issues/7)) — upstream emits `"YYYY-MM-DD HH:MM:SS+00:00"` on this endpoint only; `normalizeCommitteeReportSubresource` rewrites to `YYYY-MM-DDTHH:MM:SSZ` so callers see one shape across the surface.
