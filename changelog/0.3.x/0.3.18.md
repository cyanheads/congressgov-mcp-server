# 0.3.18 — 2026-05-20

## Fixed

- **`congressgov_member_lookup` list** ([#12](https://github.com/cyanheads/congressgov-mcp-server/issues/12)) — combining `congress` with `stateCode` (and optionally `district`) now hits `/member/congress/{congress}/{stateCode}[/{district}]` instead of throwing `validationError`. Congress.gov supports the nested path; the prior guard was wrong. `CongressApiService.listMembers` builds the path with a `/member/congress/{congress}` prefix when `congress` is set, then appends `stateCode` / `district` segments.
- **`congressgov_committee_lookup` sub-resources** ([#10](https://github.com/cyanheads/congressgov-mcp-server/issues/10)) — missing committee codes on `reports`, `bills`, `nominations`, and `house-communication` sub-resources now surface as `NotFound` with the committee identifier in `error.data`. Congress.gov returns an HTTP `500` with a Django `Committee matching query does not exist. (DoesNotExist)` body for these — `CongressApiService.tryNotFound` now classifies that pattern as `NotFound`, generalizing the same handling previously inlined only in `getCrsReport`. `isMissingEntityErrorBody` regex extended to match `does not exist` alongside `not found` / `no data`.

## Changed
