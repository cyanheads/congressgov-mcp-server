# 0.3.15 — 2026-05-08

Maintenance release. Pulls in the full 0.8.x arc of `@cyanheads/mcp-ts-core` — typed error contracts, `ctx.fail` / `ctx.recoveryFor`, DataCanvas + `spillover()`, telemetry dashboard scaffolding — plus the engine + Docker base bumps from framework 0.8.19. Tool and resource handlers are unchanged; this server doesn't yet declare `errors[]` contracts or use canvas, but the framework surface is now available to adopt incrementally.

## Added

- **Framework Antipatterns devcheck step** — new `scripts/check-framework-antipatterns.ts` (mirrored from framework 0.8.x) wired in via `scripts/devcheck.ts`. Three rules guard against SDK-coupling shortcuts in framework consumers: `inputSchema-downgrade`, `inputSchema-mutation`, `transport-text-match`. Runs standalone (`bun run scripts/check-framework-antipatterns.ts`) or as part of `bun run devcheck`.
- **`scripts/split-changelog.ts`** — synced from the framework. One-shot migration helper for the directory-based changelog convention; retained alongside `build-changelog.ts` for parity with upstream.
- **Three new skills synced from the framework**: `api-canvas` (Tier 3 SQL/analytical workspace primitive — `acquire → registerTable → query → export`, plus `spillover()` for inline-preview-with-canvas-overflow patterns), `api-telemetry` (catalog of every span, metric, attribute, completion-log field, env var, and runtime caveat the framework emits), `tool-defs-analysis` (read-only audit of MCP definition language across the surface — voice, leaks, defaults, recovery hints, output descriptions).
- **`scripts/build-changelog.ts` `security: boolean` frontmatter** — pairs with the `## Security` section to render a `🛡️ Security` badge in the rollup. Both flags render in fixed order (`· ⚠️ Breaking · 🛡️ Security`) when set. Validated like `breaking` (must be literal `true`/`false`).

## Changed

- **`@cyanheads/mcp-ts-core` `^0.7.0` → `^0.8.19`** — single jump across 19 releases. Notable framework additions now available to adopt: declarative `errors[]` contracts on tool/resource definitions with typed `ctx.fail(reason)` and `ctx.recoveryFor(reason)`, `httpErrorFromResponse` / `httpStatusToErrorCode` utilities, `partialResult` / `failureEntrySchema` typed succeeded/failed pattern, three more error factories (`internalError`, `serializationError`, `databaseError`), DataCanvas (`@cyanheads/mcp-ts-core/canvas`) with `spillover()` and `inferSchemaFromRows`. `MCP_AUTH_MODE=none` over HTTP now defaults `tenantId` to `'default'` so `ctx.state` works without minting JWTs (closes [cyanheads/mcp-ts-core#87](https://github.com/cyanheads/mcp-ts-core/issues/87)). No breaking changes; no handler edits required for this server.
- **Engines bumped:** `node` `>=22.0.0` → `>=24.0.0`, `bun` `>=1.2.0` → `>=1.3.0`, mirrored from framework 0.8.19. `packageManager` already pinned at `bun@1.3.2`.
- **Dockerfile base image** `oven/bun:1` → `oven/bun:1.3` (build + runtime stages) to align with the engines floor.
- **`@biomejs/biome` `^2.4.13` → `^2.4.14`**, **`@types/node` `^25.6.0` → `^25.6.2`**, **`tsc-alias` `^1.8.16` → `^1.8.17`** — patch bumps via `bun update --latest`.
- **`scripts/devcheck.ts`** + **`scripts/build-changelog.ts`** — resynced from the framework package. Devcheck picks up the new `Framework Antipatterns` step (between `MCP Definitions` and `Docs Sync`); `build-changelog` parses the new `security` frontmatter flag.
- **Skills refreshed from the framework package** (Phase A of `maintenance`): `add-service`, `add-tool`, `api-config`, `api-context`, `api-errors`, `api-linter`, `api-services`, `api-utils`, `api-workers`, `design-mcp-server`, `field-test`, `maintenance`, `release-and-publish`, `report-issue-framework`, `report-issue-local`, `security-pass`, `setup`. Mirrors at `.claude/skills/` and `.agents/skills/` re-propagated (Phase B). Local `devcheck` + `migrate-mcp-ts-template` skills preserved.
- **`CLAUDE.md` + `AGENTS.md`** — adopted framework-authored template updates: `api-canvas` / `api-telemetry` / `tool-defs-analysis` slotted into the skill table, changelog frontmatter section now documents `security: boolean`, references to engines floor + skill set match the new framework baseline.
- **`README.md` MCP SDK badge** `^1.27.1` → `^1.29.0` to match the version `@cyanheads/mcp-ts-core` actually pulls in.
- **`changelog/template.md`** — refreshed to the framework's current authoring guide (added `security` frontmatter slot, expanded "Audience / Tone / Cut" prose).
