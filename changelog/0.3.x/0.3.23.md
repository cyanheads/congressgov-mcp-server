# 0.3.23 — 2026-05-30

## Changed

- **`@cyanheads/mcp-ts-core` `^0.9.13` → `^0.9.16`** — framework adoption across the 0.9.14–0.9.16 series.
- **Enrichment on browse/list tools** — all 10 tool handlers now call `ctx.enrich.echo()`, `ctx.enrich.total()`, and (on empty results) `ctx.enrich.notice()`. The `enrichment` block (`effectiveQuery`, `totalCount`, `notice`) surfaces in both `structuredContent` JSON and the `content[]` markdown trailer, reaching both Claude Code and Claude Desktop.
- **`listOutput` / `listOrDetail` output schemas** — the standalone `query` field removed; replaced by the typed `enrichment: listEnrichment` block (`effectiveQuery`, `totalCount`, `notice`). Agents consuming `structuredContent` that previously read `result.query` should read `result.enrichment.effectiveQuery` instead.
- **`buildQueryEcho` → `buildEffectiveQuery`** — renamed to match enrichment field naming.

## Fixed

- **`structuredContent` output validation on list tools** — the previous `output.query` field name did not match the framework's stored key, which caused validation failures on the affected list-mode tools. The field is now carried through the `enrichment` block, which uses the correct contract.
- **Single-record get paths** — `bill-lookup`, `enacted-laws`, `member-lookup`, and `senate-nominations` were missing enrichment on their single-record fetch branches. All four now populate `ctx.enrich.echo()` and `ctx.enrich.total(1)` on the detail path.

## Dependency bumps

- `@cyanheads/mcp-ts-core` `^0.9.13` → `^0.9.16`
