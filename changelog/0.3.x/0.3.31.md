# 0.3.31 — 2026-06-20

Framework maintenance release — tracks the `@cyanheads/mcp-ts-core` 0.10.7–0.10.9 line. No tool, resource, or prompt surface changes; behavior is unchanged.

## Added

- **`scripts/check-dependency-specifiers.ts`** — new `devcheck` step (`Dependency Specifiers`, `--no-dep-specifiers`). Hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` map, catching the case where `bun update --latest` writes a literal `latest` dist-tag into the lock and lets a later `bun install` re-resolve past the `package.json` range. Static and local — runs in the default and `--fast` passes.
- **Plugin marketplace manifest checks** — `scripts/lint-packaging.ts` gains check 10, validating `.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, and `.codex-plugin/mcp.json` for non-empty descriptions and scoped/unscoped identity correctness (display fields carry the unscoped machine name; the `npx -y` install arg carries the full package name). Gated by the new `devcheck.config.json` `packaging.pluginManifests` flag (default on); the `devcheck` Packaging gate now also runs when a plugin manifest is present.

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.10.6` → `^0.10.9`. Notable upstream changes consumed on upgrade: `ctx.content` collector for non-text content blocks (image/audio bytes ride `content[]` only, never `structuredContent`); fresh-scaffold `devcheck` guards skip git-dependent and changelog-sync checks cleanly before `git init`; `check-skill-versions` skips a skill deleted from the worktree instead of crashing; `DuckdbProvider.describe({ tableName })` qualifies its pushed `WHERE` filters (fixes an ambiguous-column binder error on filtered describes); the Canvas SQL gate classifies a SELECT-shaped statement that fails to prepare (bad column/function/expression) as `invalid_sql` with the DuckDB binder detail rather than `non_select_statement`; biome `2.4.16` → `2.5.0`.
- **`devcheck.config.json`** — added `packaging.pluginManifests: true` to opt into the new plugin-manifest checks.
- **`scripts/devcheck.ts`, `scripts/build-changelog.ts`, `scripts/check-framework-antipatterns.ts`, `scripts/check-skill-versions.ts`, `scripts/lint-packaging.ts`** — re-synced to the 0.10.x framework bodies (shared `isGitRepo()` skip guard, changelog empty-tree `--check` skip, worktree-deletion guard, plugin-manifest validation).
- **Framework skills synced** to the 0.10.x bodies (`add-tool`, `api-auth`, `api-canvas`, `api-config`, `api-context`, `api-errors`, `api-services`, `api-telemetry`, `field-test`, `git-wrapup`, `orchestrations`, `polish-docs-meta`, `report-issue-local`, `tool-defs-analysis`).
