# 0.3.29 — 2026-06-04

## Fixed

- **`congressgov_committee_lookup`** now auto-resolves name-like `committeeCode` inputs (e.g. `"senate banking"`) instead of rejecting them — fetches parent committees, runs the same token-match filter as `operation:'list'`, and proceeds with the resolved code when exactly one match is found; returns candidates when zero or multiple match, with a notice guiding the caller ([#39](https://github.com/cyanheads/congressgov-mcp-server/issues/39)).
- **`committeeCode` schema** adds a `.regex()` pattern guard that rejects malformed codes at parse time (e.g. `ssbk` with no digit suffix) before any API call is made, surfacing a clear error message with recovery instructions.
- **`committeeCode` description** updated to document the name-pass-through behavior and 2-digit suffix format requirement.
